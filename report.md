## Task 4: “Два датчика” на `city_day.csv`

В качестве датасета с двумя “датчиками” использован `data/city_day.csv` (источник: Kaggle, “Air Quality Data in India (2015 - 2020)”: https://www.kaggle.com/datasets/rohanrao/air-quality-data-in-india).

Выбор “датчиков” (как двух измерений одной системы) в ноутбуке:
- город: `Delhi`
- датчик A: `PM2.5`
- датчик B: `PM10`

Сделано:
1. Загрузка двух рядов (ежедневные измерения).
2. Заполнение редких пропусков линейной интерполяцией.
3. Фильтрация (MA с окном около 31 дня + Калман) для подавления дневного шума.
4. Анализ связи между “датчиками”: корреляция Пирсона и кросс-корреляция по лагам (в днях).
5. FFT-оценка доминирующей периодичности для каждого ряда.

Численные выводы (Delhi, 2015-01-01 .. 2020-07-01, n=2009):
- Pearson corr(PM2.5, PM10) = **0.8546**
- лучшая кросс-корреляция достигается при лаге **0 дней**, то есть изменения серий в среднем синхронны

Дополнительно по рядам (по базовым метрикам и FFT):
- Средние уровни: PM2.5 ≈ **117.10**, PM10 ≈ **231.10**.
- Наблюдается слабый нисходящий тренд (trend_slope/day < 0) для обоих рядов.
- Доминирующие периоды по FFT находятся в масштабе сотен дней (порядка сезонности/годовых циклов): для PM2.5 ≈ **334.83** дней, для PM10 ≈ **182.64** дней.
